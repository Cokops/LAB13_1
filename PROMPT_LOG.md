# Prompt Log — Проект Lab13: Мультиагентная система мониторинга IT-инфраструктуры

## Описание проекта
Создание распределённой мультиагентной системы для мониторинга IT-инфраструктуры с использованием **Go** (агенты), **Python** (оркестратор, LLM-агент, веб-панель), **NATS** (брокер сообщений), **Redis** (состояние), **Jaeger** (трассировка) и **Ollama** (LLM).

---

## Этап 1: Анализ и инициализация инфраструктуры

### Промпт 1
**Инструмент:** AI Agent (DeepSeek/ChatGPT)  
**Промпт:** "Ты — эксперт по DevOps и мультиагентным системам. Напиши файл docker-compose.yml для следующих сервисов: NATS (nats:latest) с портом 4222, включи мониторинг на 8222; Redis (redis:alpine) с портом 6379; Jaeger (jaegertracing/all-in-one:latest) с портами 16686 (UI), 4317 (OTLP gRPC), 4318 (OTLP HTTP); Ollama (ollama/ollama:latest) с портом 11434, смонтируй том для моделей. Все сервисы должны быть в одной сети 'monitoring-net'. Добавь healthcheck для каждого. Напиши также команду для предварительной загрузки модели LLM (например, llama3.2:1b) через ollama."

**Результат:** Создан `docker-compose.yml` с 4 сервисами, сетью `monitoring-net`, healthcheck-ами и томом для Ollama. Также создан `README.md` с инструкциями.

## Этап 2: Разработка Go-агента

### Промпт 2
**Инструмент:** AI Agent  
**Промпт:** "Ты — Go-разработчик. Напиши универсального агента для мониторинга IT-инфраструктуры на Go. Требования:
1. Подписка на NATS-очередь 'tasks.monitoring'
2. Поддержка типов задач: 'collect_metrics', 'detect_anomaly', 'send_alert', 'auto_recovery'
3. После выполнения публикует результат в 'tasks.result'
4. Состояние (счётчик выполненных задач) хранит в Redis и восстанавливает при старте

6. Поддержка аукциона: подписка на 'tasks.auction.request', публикация ставок в 'tasks.auction.bids'
Агент должен принимать аргументы: --id, --nats-url, --redis-url, --jaeger-endpoint. Дай полный код: main.go + отдельные файлы для metrics, anomaly, alert, recovery, state, tracing, auction."

## Этап 9: Исправление ошибок импорта в тестах

### Промпт 11 (проблема при запуске)
**Инструмент:** AI Agent  
**Промпт:** "При запуске `py -m pytest tests/ -v` возникает ошибка: `ModuleNotFoundError: No module named 'llm_agent'`. В чём ошибка?"

**Анализ ошибки:** Папка называется `llm-agent` (с дефисом), но Python не может импортировать модуль с дефисом в имени. Дефис интерпретируется как оператор вычитания.

**Решения предложены:**
1. Переименовать `llm-agent` → `llm_agent`
2. Добавить `__init__.py`
3. Использовать `PYTHONPATH` или `sys.path.insert()`
4. Установить проект в editable-режиме через `pip install -e .`

**Что пришлось исправлять вручную:** потребовалось вручную переименовать папку `llm-agent` в `llm_agent` через проводник или команду `mv`  
**Время:** ~5 мин
